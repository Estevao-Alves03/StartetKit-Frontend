# Regras Gerais

Sempre utilizar:

- TypeScript.
- React funcional.
- Hooks.
- Código limpo.
- Componentes reutilizáveis.


---

# Responsividade

Todo componente deve funcionar em:

- Desktop.
- Tablet.
- Mobile.


---

# Autenticação

Não criar sistemas próprios.


Não utilizar:

- Supabase Auth.
- Firebase Auth.
- OAuth externo.
- Banco local de usuários.


O Laravel será responsável futuramente.


O frontend terá apenas:

- Tela de login.
- Formulários.
- Estados visuais.
- Rotas preparadas.


---

# Antes de criar arquivos

Verificar:

1. Já existe algo parecido?
2. Pode reutilizar?
3. Está seguindo arquitetura?
4. É realmente necessário?
