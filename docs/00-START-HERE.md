# Regras Principais do Projeto

Este documento define as regras obrigatórias para desenvolvimento do frontend.

Antes de criar ou modificar qualquer arquivo:

- Ler toda a documentação.
- Respeitar a arquitetura existente.
- Não criar novas pastas sem necessidade.
- Não alterar padrões definidos.
- Reutilizar componentes existentes.

---

# Responsabilidade do Frontend

O frontend será responsável somente por:

- Interface do usuário.
- Experiência visual.
- Comunicação com APIs.
- Estados visuais.
- Navegação.

---

# Responsabilidades do Backend

O backend Laravel será responsável futuramente por:

- Regras de negócio.
- Autenticação.
- Banco de dados.
- Permissões.
- Processamentos.

---

# Regra Principal

Antes de qualquer alteração:

1. Verificar se já existe uma solução.
2. Reutilizar componentes.
3. Evitar duplicação.
4. Manter organização.
5. Criar código preparado para integração com Laravel.
