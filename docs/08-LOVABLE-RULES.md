# Regras do Lovable

O Lovable é uma ferramenta de criação de interface.

Ele não substitui a arquitetura definida.


---

# Não criar tudo de uma vez.

Nunca gerar:

- Todas as páginas.
- Todos componentes.
- Todos módulos.


---

# Antes de implementar

Informar:

- O que será criado.
- Arquivos adicionados.
- Arquivos alterados.


---

# Prioridades

1. Organização.

2. Reutilização.

3. Manutenibilidade.

4. Responsividade.

5. Experiência do usuário.

6. Velocidade.


---

# Proibido

Não:

- Criar arquitetura paralela.
- Adicionar bibliotecas sem autorização.
- Criar autenticação própria.
- Alterar estrutura existente.
